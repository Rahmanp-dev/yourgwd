# Frontend Design Skill

Guidance for distinctive, intentional visual design when building new UI or reshaping an existing one.

## Key Methodology Summary

1. **Ground it in the Subject**: Name the concrete subject, audience, and the page's single job. Avoid generic templates.
2. **Hero as a Thesis**: Start with the most characteristic thing in the subject's world (e.g., headline, interactive element).
3. **Typography**: Pair display and body fonts deliberately, set clear type scale, make text treatments memorable.
4. **Structure is Information**: Use structural dividers, labels, and indicators to encode real truths about the content, not just for decoration.
5. **Leverage Motion Deliberately**: Target curated micro-interactions and scroll reveals; keep them subtle to avoid feeling AI-generated.
6. **Restraint and Self-Critique**: Spend boldness in one place (a single "signature" element). Keep the rest clean. No emojis, ensure keyboard focus, mobile-first responsive layout, and respect reduced motion.
7. **Writing in Design**: Write from the user's perspective, use active voice, avoid system speak, treat empty states as invitations to act.
